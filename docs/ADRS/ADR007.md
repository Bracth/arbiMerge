ADR 007: Cambio de Yahoo Finance a Finnhub para Sincronización de Precios en Tiempo Real
Fecha: 24 de marzo de 2026

Estado: Aceptado

Contexto: Las nuevas políticas de Yahoo Finance han hecho que su API sea prácticamente inutilizable para la sincronización de datos de mercado en tiempo real en este proyecto. La latencia y las restricciones de acceso impiden obtener actualizaciones de alta frecuencia, las cuales son esenciales para el cálculo preciso de los diferenciales (spreads) de arbitraje y la sincronización de precios.

Decisión: Utilizar Finnhub como el proveedor principal de datos de mercado a través de su API de WebSocket. Esto reemplaza el mecanismo anterior basado en Yahoo Finance para la obtención de precios en tiempo real.

Consecuencias:

Positivas:
* Mejor precisión en los cálculos de spread al contar con datos actualizados al instante.
* Baja latencia en la entrega de precios desde el backend hacia el frontend.
* Escalabilidad para monitorear más símbolos sin las limitaciones de cuotas de las APIs REST tradicionales de Yahoo Finance.

Negativas:
* Requiere la implementación de lógica para la gestión de conexiones de WebSockets (reconexiones, latidos de corazón) en el backend.

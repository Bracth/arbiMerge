ADR 002: Uso de Node.js + WebSockets para el Backend
Fecha: 18 de marzo de 2026
Estado: Aceptado

Contexto: La aplicación requiere ingesta de datos financieros en tiempo real y transmisión de baja latencia a múltiples clientes simultáneos. Evaluamos Python (FastAPI) frente a Node.js. El equipo de desarrollo tiene mayor dominio de JavaScript/TypeScript.

Decisión: Utilizar Node.js con librerías de WebSockets (ej. Socket.io o ws) implementando un patrón de Publicador/Suscriptor (Pub/Sub) centralizado en nuestro servidor para no saturar los límites de la API financiera externa.

Consecuencias:

Positivas: El Event Loop asíncrono de Node.js es ideal para manejar problemas I/O Bound con miles de conexiones abiertas; la curva de aprendizaje y desarrollo será mínima debido al expertise previo del equipo.

Negativas: Si en fases futuras se requiere procesamiento matemático pesado (ej. simulaciones de Monte Carlo), el rendimiento en un solo hilo (single-thread) podría ser un cuello de botella en comparación con Python.

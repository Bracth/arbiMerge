ADR 003: Contenerización y Despliegue con Docker
Fecha: 18 de marzo de 2026
Estado: Aceptado

Contexto: La organización de la hackathon proveerá la infraestructura de hosting (tipo VPS), por lo que servicios de plataforma gestionada (PaaS) como Vercel o Supabase no son viables para el entorno de producción final del evento.

Decisión: Empaquetar todos los servicios (Frontend, Backend y Base de Datos) utilizando Docker y orquestarlos a través de Docker Compose.

Consecuencias:

Positivas: Eliminación del problema "en mi máquina sí funciona"; portabilidad total entre el entorno de desarrollo y el servidor del evento; despliegue unificado con un solo comando (docker compose up).

Negativas: Añade una ligera capa de complejidad en la configuración inicial de las redes internas entre contenedores.

ADR 004: Sustitución de Next.js por React (Vite) + Nginx
Fecha: 18 de marzo de 2026
Estado: Aceptado

Contexto: Aunque Next.js es un estándar para aplicaciones React modernas, su integración con WebSockets personalizados dentro de un entorno Dockerizado requiere levantar un "Custom Server", rompiendo muchas de sus optimizaciones nativas y complicando el despliegue rápido.

Decisión: Utilizar React empaquetado con Vite para el desarrollo del frontend. En producción (Docker), los archivos estáticos generados serán servidos por un servidor web ultra-ligero (Nginx).

Consecuencias:

Positivas: Soporte fluido y nativo para conexiones WebSocket; el contenedor de Nginx consumirá mínimos recursos (RAM/CPU); tiempos de compilación y recarga en desarrollo ultra rápidos gracias a Vite.

Negativas: Pérdida del Server-Side Rendering (SSR) y optimización SEO nativa de Next.js (lo cual es aceptable, ya que ArbiMerge es un dashboard privado, no un blog público).
